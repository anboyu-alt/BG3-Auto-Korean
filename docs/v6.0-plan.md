## ③ UI 완전 다국어화(15개 언어, 게임 언어 자동) + 로그 영어화  (먼저 / 가장 중요)
원래 "UI는 영어인데 로그가 한국어" + "영어 기본" 지적 → 확장: **UI를 15개 언어 전부 지원하고
게임 언어에 자동 맞춤.** 그러면 "영어 기본값"이 불필요(영어는 폴백일 뿐).

### 3-0. 15개 언어 UI (신규, 이 기능의 핵심)
- i18n 로더 확장: 현재 en/ko만 → 15개 코드 매핑(fr de es pl ru zh tr pt-BR it es-LA zh-TW uk ja) +
  **누락 키는 en 폴백**. (`bg3gui/i18n/__init__.py` load())
- 13개 언어 STRINGS 파일 필요(현 en.py/ko.py 외). UI 문자열 ~80~100개 + desc.* 도움말.
- **생성 방법: 이 앱의 Gemini로 en.py를 13개 언어로 자동 번역하는 빌드 스크립트**(번역기가 자기 UI를 번역).
  기술 용어(PAK, Divine.exe, MCM, LSTag, API 등)는 번역 금지로 프롬프트에 명시.
- AI 번역 UI는 "기계번역, 커뮤니티 수정 환영"으로 표기 → 원어민 PR/제보로 개선.
- 유지보수: UI 문자열 변경 시 스크립트 재실행으로 15개 재생성.
- 폰트: 15개 중 RTL/특수 스크립트 없음(이미 CJK 렌더됨) → 문제없음.

### 3a. 기본 언어 자동 설정 (게임 언어 기반) — ✅ 검증 완료
- `bg3core/config.py`: `app_language: str = "ko"` 가 기본.
- **최선책(검증됨): BG3 `language.lsx`로 자동 감지.** `…\Data\Localization\language.lsx`에
  `<attribute id="MapKey" value="Language"/>` + `<attribute id="Value" value="Korean"/>` 형태로
  게임 언어가 명시됨(실측 확인). 이 Value가 우리 LANGUAGE_PROFILES folder_name과 일치("Korean").
  - 첫 실행(config 없음) 시: language.lsx의 Language 값 → **기본 target_language** 설정,
    그리고 **app_language = (Korean이면 ko, 아니면 en)**. 대상 언어 + UI 언어를 한 번에 자동화.
  - 저장된 사용자 설정은 보존(첫 실행 한정 기본값 제안).
- 폴백: BG3 경로 미설정/파일 없음 → 기본 "en"(또는 OS 로캘). 
- 주의: 앱 UI는 ko/en 2종뿐 → 비한국어 게임은 대상만 해당 언어, UI는 영어. 이름 매핑은
  대부분 1:1이나 중국어 변형 등 일부 별칭 맵 + 폴백 필요. **BG3 경로는 ④와 공유** → 함께 구현.

### 3b. 로그 영어화  (핵심 작업, 기계적)
- 한국어 로그가 하드코딩된 파일(실제 로그 대상):
  - `bg3core/pipeline.py` (~45), `bg3core/divine.py` (~30), `bg3core/repair.py` (~23),
    `bg3core/mcm/loca_handles.py` (~26), `mcm/patch_builder.py` (~24), `mcm/lua_handler.py` (~15),
    `mcm/blueprint.py` (~15), `mcm/whitelist.py` (~12), `mcm/__init__.py` (~8),
    `bg3core/reviewer.py`, 그리고 `bg3core/translate.py` 의 `_log(...)` 부분.
- **영어화 대상이 아닌 것(건드리지 말 것):**
  - `bg3core/glossary.py` (391줄) = **용어집 데이터(한국어 번역)**. 콘텐츠라 유지.
  - `bg3core/language.py` (~39) = 언어 표시명/프롬프트명 데이터. 유지.
  - `bg3core/translate.py` 의 **시스템 프롬프트(한국어로 Gemini에 지시)** = UI 아님. 유지(번역 품질용).
- 방식 선택:
  - **A(추천): 로그 영어 단일화** — 로그는 항상 영어. core에 i18n 배선 불필요. 국제 배포 표준.
  - B: 로그도 i18n(ko/en) — core에 translator 주입 필요, 작업량 큼. 비추천.
- 작업법: `grep -rnP "[가-힣]" bg3core/{pipeline,divine,repair,reviewer}.py bg3core/mcm/*.py` 로 뽑아
  각 `_log/logger/print` 한국어 문자열을 영어로 치환(포맷 인자 `{}` 유지).
  - 예: "총 블록: {} (고유: {}, 중복 제거: {})" → "Total blocks: {} (unique: {}, deduped: {})"
  - "언팩 중" → "Unpacking", "번역 시작..." → "Translating...", "청크 수" → "Chunks",
    "API 필요" → "Need API", ".loca 생성" → "Generating .loca", "리팩 중" → "Repacking",
    "번역 완료" → "Translation complete", "이미 번역된 파일. 스킵" → "Already translated, skipping" 등.
- 테스트: 한국어 로그 문자열을 assert하는 테스트가 있는지 확인 후 함께 갱신
  (`grep -rnP "[가-힣]" tests/`). 대부분 없을 것으로 예상.
